# Source data

This directory holds maintainer-edited inputs for the data build pipeline. See the root [README](../../README.md) for script usage.

## Files

| File                         | Purpose                                                                                         |
| ---------------------------- | ----------------------------------------------------------------------------------------------- |
| `institutions.json`          | Canonical institution records (names, locations, reciprocity tiers, contacts, geocoding fields) |
| `.cache/aza-find-a-zoo.html` | Fetched at build time from AZA's directory page (gitignored)                                    |
| `us-zip-centroids.csv`       | US ZIP code → latitude/longitude centroids for straight-line distance estimates                 |
| `match-report.json`          | Generated by `yarn data:build` — review unmatched website entries                               |

## Provenance

### AZA institution and reciprocity data

Institution names, reciprocity tiers, and contact information are derived from publicly available AZA materials, primarily the [AZA Reciprocity Chart](https://www.aza.org/reciprocity) (last synced 2026-06-18).

**ZooCompass is not affiliated with, endorsed by, or sponsored by the Association of Zoos and Aquariums (AZA).** AZA names, logos, and institution listings remain the property of their respective owners.

Guest-services phone numbers and generic inbox emails may be shown in the app when present in source data.

### AZA Find a Zoo directory fetch

Website URLs are attached by fetching AZA's live [Find a Zoo or Aquarium](https://www.aza.org/find-a-zoo-or-aquarium) page at build time. The HTML is cached locally under `.cache/` (not committed).

```bash
yarn data:fetch-aza   # fetch and cache only
yarn data:build       # fetch if cache is missing, then build
yarn data:build -- --refresh-aza   # force a fresh fetch
yarn data:build -- --offline       # require existing cache
```

### ZIP centroids

`us-zip-centroids.csv` provides approximate geographic centers for US ZIP codes. The app uses these for straight-line distance estimates only — not driving directions.

If you replace this file, document the new source in this README and run `yarn data:build-zips`.

## Refreshing data

```bash
# After editing institutions.json
yarn data:build

# After editing us-zip-centroids.csv
yarn data:build-zips

# Optional: geocode missing addresses (requires NOMINATIM_EMAIL in .env)
yarn enrich:addresses
```
